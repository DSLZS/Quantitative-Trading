## 一、问题诊断

### 1.1 原始问题

在 V3.0 首次回测中发现致命错误：
```
WARNING: 预测时特征数不匹配：期望 30, 实际 16
```

**根本原因**: 预测时未正确复用训练时的特征转换流程，导致：
- 正交化矩阵未保存
- 特征选择列名未持久化
- 合成特征在推理时丢失

### 1.2 问题影响

| 组件 | 训练时 | 预测时 (Bug) | 影响 |
|------|--------|-------------|------|
| 原始因子 | 16 | 16 | ✓ |
| 正交化后 | 16 | 16 | ✓ |
| 合成特征 | 136→30 | 16 (丢失) | ✗ |
| 最终输入 | 30 | 16 | ✗ |

## 二、修复方案

### 2.1 特征工程管线修复 (Inference Consistency)

#### 修复前代码问题
```python
# ❌ 预测时重新计算可用列，与训练时不一致
available_cols = [c for c in base_columns if c in df.columns]
```

#### 修复后代码
```python
# ✓ 保存训练时的特征列
self.training_feature_columns = available_cols.copy()

# ✓ 预测时严格使用训练时的列
feature_cols = self.training_feature_columns
available_cols = [c for c in feature_cols if c in df.columns]
```

### 2.2 核心组件状态持久化

| 组件 | 持久化状态 | 用途 |
|------|-----------|------|
| `FactorOrthogonalizer` | `mean_`, `std_`, `transformation_matrix` | 正交化转换 |
| `FeatureSynthesizer` | `selected_features`, `input_columns` | 特征选择 |
| `StandardScaler` | `mean_`, `scale_` | 数据标准化 |

### 2.3 三屏障碍法优化

| 参数 | 修复前 | 修复后 | 理由 |
|------|--------|--------|------|
| 时间障碍 | 5 天 | **12 天** | 延长持有期过滤噪声 |
| 上轨阈值 | 2.0σ | **2.5σ** | 提高止盈门槛 |
| 下轨阈值 | 2.0σ | **1.5σ** | 降低止损门槛 (非对称) |

**非对称阈值设计逻辑**:
```
止盈：+2.5σ (需要更强的上涨信号)
止损：-1.5σ (更快地截断亏损)
```

### 2.4 GMM 状态增强

新增状态波动率记录：
```python
self.state_volatility[state_id] = float(np.std(y_state))
```
